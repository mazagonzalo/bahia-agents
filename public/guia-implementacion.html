Bahía Agents

12 pasos para activar los 5 agentes de marketing IA. El código ya está listo en GitHub — solo hay que conectar los servicios.

Pasos 12
Tiempo estimado ~2h
Costo mensual ~$100
Servicios externos — conseguir credenciales
01
Inteligencia IA
Anthropic — Claude API
5 min Pay per use
1
Ve a console.anthropic.com e inicia sesión
Si no tienes cuenta, crea una con tu correo
2
En el sidebar: API Keys → Create Key
Nombre sugerido: bahia-agents-prod
3
Copia la key que empieza con sk-ant-
4
Agrega créditos en Settings → Billing
Mínimo $20 USD para arrancar cómodo
Variable a guardar 
ANTHROPIC_API_KEY=sk-ant-api03-XXXXXXXXXXXXXXXXXXXX
02
Base de datos
Supabase
10 min Free tier
1
Ve a supabase.com → New Project
2
Configura el proyecto
— Nombre: bahia-agents
— Contraseña: anota una segura (la vas a necesitar)
— Región: East US (N. Virginia) o la más cercana
3
Espera ~2 minutos a que el proyecto inicie
4
Ve a Settings → API y copia:
— Project URL → guarda como SUPABASE_URL
— En Project API Keys: copia service_role secret → guarda como SUPABASE_SERVICE_KEY
— No uses la anon key — el agente necesita acceso completo
i La base de datos se puebla en el Paso 10 con el schema SQL. Aquí solo obtienes las credenciales.
Variables a guardar 
SUPABASE_URL=https://xxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.XXXXX

04
Canal de comunicación
Meta for Developers — WhatsApp
30–45 min Gratis (primera 1,000 conv/mes) El más largo
4a — Crear la app en Meta
1
Ve a developers.facebook.com
Necesitas una cuenta de Facebook personal asociada a una Business account
2
My Apps → Create App
3
Selecciona tipo: Business
4
Nombre: Bahia Agents → asocia tu Business account
5
En el dashboard de la app: Add Product → WhatsApp
4b — Obtener credenciales de WhatsApp
6
WhatsApp → API Setup
7
Copia el Temporary access token
Guarda como WHATSAPP_TOKEN
Nota: caduca cada 24h en modo dev. En producción usa un System User Token permanente.
8
Copia el Phone number ID
Guarda como WHATSAPP_PHONE_ID
9
Inventa un Verify Token secreto
Puede ser cualquier string, ej. bahia-webhook-2024
Guarda como WHATSAPP_VERIFY_TOKEN
10
Tu número de WhatsApp del admin
Formato internacional sin +: 521XXXXXXXXXX
Guarda como ADMIN_PHONE
4c — Token permanente (producción)
11
Business Settings → System Users → Add
Nombre: bahia-agents-bot | Rol: Admin
12
Generate Token → selecciona la app → permisos: whatsapp_business_messaging
Este token no caduca. Úsalo como WHATSAPP_TOKEN final.
Variables a guardar 
WHATSAPP_TOKEN=EAAxxxxxxxxxxxxxxxxxxxxxxxxxx
WHATSAPP_PHONE_ID=123456789012345
WHATSAPP_VERIFY_TOKEN=bahia-webhook-2024
ADMIN_PHONE=521XXXXXXXXXX

08
Hosting + API routes + Crons
Vercel — Deploy
15 min $20/mes (Pro requerido para crons)
1
Ve a vercel.com → inicia sesión con GitHub
2
New Project → Import el repo mazagonzalo/bahia-agents
3
Framework: Next.js (lo detecta automáticamente)
4
ANTES de hacer deploy: expande Environment Variables
Agrega todas las variables del .env.local una por una, o usando la opción de importar desde archivo
5
Haz clic en Deploy
El primer deploy tarda ~2-3 minutos
6
Copia la URL del deployment (ej. bahia-agents.vercel.app)
7
Regresa a Settings → Environment Variables y agrega:
NEXT_PUBLIC_URL = https://bahia-agents.vercel.app
8
Redeploy para que tome la variable NEXT_PUBLIC_URL
Deployments → último deployment → los tres puntos → Redeploy
9
Activa plan Pro para habilitar los cron jobs
Settings → Billing → Upgrade to Pro ($20/mes)
Los crons del vercel.json se activan automáticamente
Ruta Schedule Qué hace
/api/cron/lead-followup 0 */4 * * * Cada 4 horas — follow-up de leads inactivos
/api/cron/daily-trends 0 8 * * * 8am diario — análisis de tendencias + carrusel
/api/cron/weekly-report 0 9 * * 1 Lunes 9am — reporte semanal al admin
09
Conectar mensajería
Configurar Webhook de WhatsApp
5 min
1
Ve a developers.facebook.com → Tu App → WhatsApp → Configuration
2
En la sección Webhook → haz clic en Edit
3
Ingresa la URL del webhook y el verify token
Callback URL: https://bahia-agents.vercel.app/api/webhooks/whatsapp
Verify Token: el valor de WHATSAPP_VERIFY_TOKEN (ej. bahia-webhook-2024)
4
Haz clic en Verify and Save
Meta va a llamar al endpoint GET para verificarlo. Si el deploy está bien, dirá "Verified"
5
En Webhook Fields: activa messages
Haz clic en Subscribe junto al campo messages
✓ Si el webhook se verificó correctamente, el Agente de Ventas ya puede recibir mensajes de WhatsApp.
10
Base de datos
Crear tablas en Supabase
5 min
1
Ve a app.supabase.com → Tu proyecto → SQL Editor
2
Haz clic en New Query
3
Abre el archivo bahia-agents/supabase/schema.sql en tu computadora y copia todo su contenido
4
Pega el SQL en el editor → haz clic en Run
5
Verifica en Table Editor que existan estas tablas:
leads · conversations · agent_memory · creatives · trends
i Si hay error con la extensión vector, actívala en: Database → Extensions → busca vector → Enable
Tabla Para qué sirve
leads Registro de prospectos: teléfono, nombre, status, score
conversations Historial de mensajes por lead (memoria del Agente de Ventas)
agent_memory Aprendizajes de los agentes: qué funcionó, qué no
creatives Carruseles y anuncios generados: borrador → aprobado → publicado
trends Tendencias detectadas diariamente en Riviera Nayarit
Verificación — probar que todo funciona
11
QA
Pruebas de cada agente
20 min
Prueba 1 — Agente de Ventas (WhatsApp)
Desde cualquier teléfono: manda "Hola, me interesa Bahía" al número de WhatsApp Business
Debes recibir respuesta de Claude en menos de 10 segundos
Verifica en Supabase → Table Editor → leads: debe aparecer tu número
Verifica en conversations: debe haber 2 registros (user + assistant)
Prueba 2 — Agente Secretaria (admin)
Desde el número del admin: manda "leads de hoy" al número de WhatsApp Business
La Secretaria debe responder con el conteo de leads del día
Manda "qué tendencias hay hoy" → debe delegarlo al Agente de Tendencias
Prueba 3 — Agente de Tendencias (manual)
Abre en el navegador: https://bahia-agents.vercel.app/api/agents/tendencias
Debe devolver JSON con tendencias detectadas
Verifica en Supabase → trends: deben haber nuevos registros
El admin debe recibir un WhatsApp con las tendencias
Prueba 4 — Vercel Logs (si algo falla)
Vercel Dashboard → Tu proyecto → Functions
Ahí puedes ver los logs de cada request y los errores
Los errores más comunes son variables de entorno mal configuradas
12
Automatización
Verificar cron jobs
5 min Requiere Vercel Pro
1
Ve a Vercel Dashboard → Tu proyecto → Cron Jobs
Deben aparecer los 3 crons del vercel.json
2
Haz clic en Trigger en el cron de daily-trends para ejecutarlo manualmente
Verifica que el admin recibe WhatsApp con tendencias del día
3
Haz clic en Trigger en el cron de lead-followup
Si no hay leads de prueba, el cron devuelve {"followedUp": 0} — eso es correcto
✓ Si los 3 crons aparecen en Vercel y se ejecutan sin error, el sistema está completamente activo y automatizado.
