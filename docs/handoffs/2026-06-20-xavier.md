## Hallazgos clave de Trainingym (para el pitch y el build)
- **No tiene API pública.** Ruta de datos = **export Excel del panel admin**. Side-doors: **Stripe propio** (Stripe Connect → todo el dinero) y **Alegra**. Webhooks existen pero angostos (2 eventos).
- **Su debilidad es la analítica** (reviewers piden "análisis más profundo"; sacar data es a mano). **Validación:** existe **FitnessKPI** (Gold's Gym/Planet Fitness) que vende justo esto encima de Trainingym.
- **Nuestro diferenciador:** agentes gobernados + WhatsApp que **cierran el loop con acción**, no solo dashboards. + Spanish/MXN-native + Motor Day-Pass.
- **Norte analítico:** frecuencia de visita temprana es la señal #1 de churn; Trainingym no hace cohortes/funnel/scoring explicable/RFM → nuestros huecos.

## Decisiones (log)
1. Objetivo #1 = **Day-pass → socio** (cuando haya datos).
2. **Multi-tenant desde el día 1** (`clubId` + RLS) → revendible (como FitnessKPI).
3. Finanzas = ingresos/MRR + cobranza/morosos + P&L/forecasting. RH = entrenadores + horarios/asistencia.
4. Agente Informe = **ambos** (reporte + persiste). Enfoques: general/retención/day-pass/finanzas-operación.
5. **🔄 PIVOTE:** sin datos todavía → **fundación-primero + demo para el pitch**; históricos = switch posterior.
